# Tavern App — 使用与查看指南

## 快速启动

### 1. 配置 API Key

在项目根目录创建 `.env.local`：
```
DEEPSEEK_API_KEY=sk-你的key
```

或设置系统环境变量 `DEEPSEEK_API_KEY`。

### 2. 启动

```bash
pnpm dev
```

浏览器打开 **http://localhost:3000**。

> 双击 `start.bat` 也可一键启动（会自动检测 pnpm 和 API Key）。

### 3. 验证连通性

```
http://localhost:3000/api/debug/ping
```

返回 `{"ok":true}` 即表示连通。若失败，访问 `/api/debug/env` 查看 key 是否被正确读取。

## 页面指南

### 主页面 `/` — AI 角色扮演聊天

```
┌──────────┬──────────────────────┬──────┐
│ 侧边栏    │      聊天区           │ 调试  │
│          │                      │      │
│ 角色列表  │  消息气泡（Markdown）  │ 浮动  │
│ 导入角色  │  输入框               │ 按钮  │
│ 预设/世界书│  RuntimeConsole     │      │
│ 正则/AN  │  StatusBar           │      │
└──────────┴──────────────────────┴──────┘
```

- **发送消息** → 底部 StateBar 显示状态：就绪 → 连接中 → 思考中 → 生成中 → 完成
- **出错** → 屏幕底部弹出红色 ErrorToast，同时 StateBar 变红
- **Prompt 调试** → 右下角 🔍 按钮，查看 AST 节点树和构建流程

### Workflow Demo 页面 `/workflow-demo` — AI Runtime 平台

**三栏布局**：

```
┌───────────┬────────────────────┬──────────────┐
│ 左侧面板   │      聊天区         │  右侧检查器   │
│           │                    │              │
│ 会话列表   │  消息气泡           │ [日志|时间线  │
│ 🧠 记忆   │  输入框             │  |节点图|Token]│
│ ⚙ 能力配置 │                    │              │
│ 📦 模板库  │                    │              │
└───────────┴────────────────────┴──────────────┘
```

#### 左侧面板

| 模块 | 功能 |
|------|------|
| **会话列表** | 新建/切换/删除会话。每个会话独立记忆和目标 |
| **🧠 长期记忆** | 展开查看角色所有记忆，含重要性和标签 |
| **⚙ 能力配置** | 勾选 emotion/memory/narrative/goal/world 开关 |
| **📦 模板库** | 两个 Tab：模板浏览 / 能力说明 |

**能力配置** 中勾选/取消能力 → 实时重组执行图 → 只有选中的节点运行。

#### 右侧检查器（4 Tab）

| Tab | 查看内容 |
|-----|---------|
| **日志** | 实时流水：`[情绪] 分析消息情绪... → happiness=+5` |
| **时间线** | 节点耗时柱状图 + 事件瀑布（点击展开详情） |
| **节点图** | 按层级排列的彩色节点：🟢完成 🔴错误 ⚪跳过。点击节点看详情 |
| **Token** | Token 来源分解（情绪/记忆/叙事/历史占比）+ 耗时瓶颈分析 |

## 如何调试

### 1. 环境问题

```
http://localhost:3000/api/debug/env    → key 是否配置
http://localhost:3000/api/debug/ping   → DeepSeek 是否连通
http://localhost:3000/api/debug/chat   → 非流式聊天是否正常
```

### 2. 聊天不通

- 看底部 **StatusBar**：红色 = 出错，鼠标悬停看错误消息
- 看 **ErrorToast** 弹窗：会给出具体解决步骤
- 看终端日志：`[SSE]` / `[ERROR]` 前缀的输出

### 3. Workflow 不执行

- 切换到「日志」Tab → 看是否有 `[GRAPH]` 日志
- 切换到「节点图」Tab → 看哪些节点 🟢 哪些 ⚪
- 切换到「Token」Tab → 看是否有实际 Token 消耗

### 4. 性能问题

- **Token Tab** → 耗时分布 → 找到 `瓶颈: xxx` 行
- 如果「模型调用」占 >90% → 正常（网络延迟）
- 如果其他节点耗时异常 → 检查节点实现

| 模式 | 适合 | 节点数 | Token |

## 聚合数据查看（右侧面板）

每次发送消息后，四个 Tab 自动更新：

### 日志 Tab
实时显示每个节点的执行状态、输出摘要、情绪变化：
```
⚙ [系统] 当前模式: 情感对话
💭 [情绪] 分析消息情绪... 好感 +3
🧠 [记忆] 检索长期记忆... 匹配 4 条
📝 [Prompt] 构建 Prompt AST... 850 tokens
🤖 [模型] 请求 DeepSeek... 输出 240 字符
✅ [完成] 响应结束 (2340ms)
```

### 时间线 Tab
- 上面：节点耗时柱状图
- 下面：事件瀑布，每条可点击展开

### 节点图 Tab
- 彩色节点按执行层级排列
- 🟢 = 完成，🔴 = 出错，⚪ = 跳过
- 点击节点 → 显示输入/输出/情绪/决策

### Token Tab
- Token 消耗总览（Prompt / Output / 总计）
- 来源分解（系统提示词 / 历史消息 / 情绪注入 / 记忆注入等）
- 耗时分布（哪个节点最慢）
